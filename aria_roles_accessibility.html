WAI-ARIA Categories: Roles, Properties, States

This page outlines WAI-ARIA categories and their purposes for accessibility. There are 3 categories...

Additional html attributes types that can be applied to elements to provide additional semantics and improve accessibility:

  1. Role Category:
    • Describes what an element is...
  2. Property Category:
    • Labels, relationships, additional semmantic info
  3. States Category:
    • Describe whether is active, selected, checked, hidden, etc...

1. Document Structure Roles

Define content as a distinct document or region to aid navigation and understanding.

  • document
  • article
  • region

2. Widget Roles

Represent interactive elements that users can manipulate. Communicate behavior to assistive technology.

  • button
  • checkbox
  • radio
  • slider
  • textbox
  • combobox
  • tab
  • menuitem

3. Landmark Roles

Identify key page regions for quick navigation.

  • banner (header)
  • navigation
  • main
  • complementary (sidebar)
  • contentinfo (footer)
  • search

4. Live Region Roles

Mark areas where content updates dynamically. Assistive technologies announce updates.

  • status
  • alert
  • log
  • marquee

5. Window Roles

Represent top-level UI windows or overlays. Define focus and interaction scope.

  • dialog
  • alertdialog
  • application

6. Abstract Roles

Conceptual roles used in the specification to organize other roles. Do not use directly in HTML.

  • range (group of slider, spinbutton, scrollbar)
  • select (group of listbox, tree, grid)
  • widget
  • structure

WAI-ARIA enhances accessibility for dynamic and interactive content. Aria communicates properly the assistive technologies

  • Such as: screen readers, screen magnifiers, speech recognition magnifiers, braille displays...

All roles shown are purely theoretical; no interactive examples included. Use semantic HTML whenever possible.
