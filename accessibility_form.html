Accessible Form in HTML:

Form with accessibility compliance:

avoid unnecessary questions

  • minimize questions per screen and limit number of tasks presented to the user
    • faster load times and reduce cognitive download

provide context clues

chunking for cognitive disability
breaking a form into steps
  • remind user of form purpose
  • clearly label the current step
  • how many tasks have been completed?
  • how many steps remain?

group related questions

  • reduce cognitive load

use clear input labels. not placeholder text.

make focus indicators obvious

  • for mobile screens, labels should go above the input, not besides it

help users spot and correct errors

  • input assistance
