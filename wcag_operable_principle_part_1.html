operable principle for web content accessibility compliance

operable

user interface components and navigation must be operable.

  • keyboard navigation.
  • Obvious focus indicators.
  • Allow enough time.
  • Disable distracting content.
  • Careful with flashing animations.
  • Help users navigate.
  • Descriptive page <title>
  • Make the link's purpose clear
  • Semantic heading structure
  • Provide alternatives to complex interactions
  • Generous target sizes.

keyboard accessible control navigation

  • Removing barriers between user and content.
  • Enable continuous keyboard navigation
  • Permanent disability, termporary, situational related to vision or motor skills.
  • All elements should be accessible using the Tab key in the keyboard.
  • Users need to be able to exit each interface element as well.
  • css focus styling is essential to make keyboard interaction accessible.
keyboard traps:
Occurs when a user can get into a component or element on a web page by using the keyboard but cannot get out of that component or element through the use of the keyboard

keyboard navigation

  • make all functionality available from a keyboard
  • enough time: provide user enough time to read and use content
  • seizures and physical reactions: do not design content in a way that is known to cause seizures or physical reactions
  • navigable: provide ways to help users navigate, find content, and determine where they are.
  • input modalities: make it easier for users to operate functionality through various inputs beyond keyboard

obvious focus indicators

  • content on hover or focus visually obvious
  • any element indicator selection hover must remain visible
  • navigate a site with focus insidcators using a keyboard
  • logical order while tabbing focus

allow enough time

  • tasks have a time limit
  • enabling and disabling time limit use for users

disable distracting content

  • animation from interactions
  • enable and disable distracting animated content

careful with flashing animations

  • seizures and physical reactions and triggers

help users navigate

  • navigable
  • users should be able to navigate
    bypass blocks
    mechanism should be available to skip blocks of content

  • lengthy navigation
